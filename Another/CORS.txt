Что такое CORS Правило одинакового источника (Same-Origin Policy) 
Запрещает делать http запросы с других источников
Браузеры распространили его и на JavaScript-запросы: по умолчанию можно получить доступ к ресурсам только из одинакового источника.
Разрабатывая бэкенд, мы, чтобы разрешить запросы из других источников, можем добавить в HTTP-ответ дополнительные заголовки, начинающиеся с Access-Control-*. 
На основе значений этих CORS-заголовков браузер сможет разрешить определённые запросы из других источников, которые обычно блокируются правилом одинакового источника.

уществует несколько CORS-заголовков, но браузеру нужен всего один из них, чтобы разрешить доступ к ресурсам из разных источников —Access-Control-Allow-Origin. 
Его значение определяет, из каких источников можно получить доступ к ресурсам на сервере.
response.setHeader(
	"Access-Control-Allow-Origin", "сайт"
);
Allowed Origins это разрешение каким сайтом можно дать доступ
В качестве значения разрешённых источников CORS позволяет указать спецсимвол *. Он означает, что доступ к ресурсам открыт из всех источников, поэтому используйте его с осторожностью.

Ещё один довольно распространённый заголовок — Access-Control-Allow-Methods. С ним будут разрешены только те запросы из других источников, которые выполнены с применением перечисленных методов.

Запрос считается простым, если в нём используются методы GET и POST и нет никаких пользовательских заголовков. 
Любые другие запросы (например, с методами PUT, PATCH или DELETE) — предварительные.
